PokeClaw ができるまで

Gemma 4 が火曜日にリリースされました。木曜日には、それを使ってスマホを操作する動作するアプリが完成していました。二晩の徹夜、一人の開発者、そして大量の Claude Code で。

作る前に何ヶ月も探し続けました。見つかったオープンソースのスマホ自動化プロジェクトは、クラウド LLM が必要だったり、スクリーンショットベースのビジョン（壊れやすい）を使っていたり、動作する APK が配布されていなかったりしました。ライブアクセシビリティツリーに対してネイティブのツール呼び出しでローカル推論を実行するものは一つもありませんでした。

PokeClaw はソロ開発者がゼロから作ったプロトタイプです。磨き込まれた消費者向けアプリではありません。あなたのデバイスで動けば最高です。動かなければ、Issue を歓迎します。

コードベース全体はオープンです。読んで、フォークして、その上に構築してください。それがこのプロジェクトの目的です。

よくある質問

どのスマホに対応していますか？

arm64 アーキテクチャの Android 9 以降に対応しています。2B モデルには約 4 GB の空き RAM が必要です。エントリー向けスマホでも動作しますが、CPU では約 45 秒のウォームアップが発生します。ML チップ搭載のフラッグシップ（Pixel 8+、Galaxy S24、Dimensity 9200+）は GPU アクセラレーションで大幅に高速化されます。

本当に無料ですか？何か裏がありますか？

裏はありません。Apache 2.0 ライセンスの完全オープンソースです。広告なし、データ収集なし、プレミアムプランなし、サブスクリプションなし。コードベース全体が GitHub にあります。ソースから自分でビルドすることもできます。

Google Assistant と何が違いますか？

Google Assistant は Google と連携した限られたアプリとしか連携できず、インターネットが必要です。PokeClaw はアクセシビリティサービスを通じてスマホ上の任意のアプリを操作でき、完全にオフラインで動作します。画面上の実際の UI 要素を読み取り、タップ操作を自分で判断します。

オフラインで使えますか？

はい。2.6 GB のモデルを一度ダウンロードすれば、すべてスマホ上で動作します。PokeClaw のインターネットアクセスを完全にブロックしても動作します。ローカルモードではネットワーク通信はゼロです。

データは安全ですか？

モデルはすべてデバイス上で動作します。会話内容、メッセージ、画面の内容はスマホの外に出ることはありません。サーバーなし、テレメトリーなし、アプリ内アナリティクスなし。ソースコードを公開していますので、ご自身で確認できます。

クラウドモデルも使えますか？

はい。PokeClaw は OpenAI 互換 API（GPT-4o、Claude、Gemini など）に対応しています。設定画面でご自身の API キーを入力してください。複雑なタスクにはクラウドモードの方が賢く対応できます。会話の途中でローカルとクラウドを切り替えることもできます。

Samsung がマルウェアと判定するのですが？

サイドローディング時の誤検知です。Samsung は Play ストア以外からインストールされた未署名の APK にはすべてフラグを立てます。このアプリは完全オープンソースです。データ収集ゼロ、ローカルモードでのネットワーク通信ゼロ。ソースからビルドして確認することもできます。

スマホで AI を試してみませんか？

APK をダウンロードして、権限を一つ許可するだけ。あとは自然な言葉でスマホを操作できます。登録不要、API キー不要、月額費用なし。
